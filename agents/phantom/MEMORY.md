# Phantom — persistent memory

Notes that should always be in Phantom's working memory, regardless of conversation history. Keep this file under a few KB; everything written here is on every turn.

For longer-form memory (transcripts, retrievable knowledge), use the SQLite memory store, not this file.

## Examples to seed with

- Your owner's name and time zone (ask them — don't assume)
- The people your owner works with, and their roles
- Daily routine constants Phantom should know about
- Standing preferences ("always confirm before sending email", etc.)
